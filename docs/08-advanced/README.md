# 08. 进阶能力

进阶能力应在核心模块、依赖注入、请求生命周期和测试已经熟练后学习。

| 能力 | 适用场景 | 学习重点 |
| --- | --- | --- |
| WebSocket | 双向实时通信 | Gateway、连接状态、鉴权、水平扩展 |
| Microservices | 事件驱动或跨进程通信 | Transport、消息模式、幂等、超时与重试 |
| Queue | 异步任务、削峰、失败重试 | 任务状态、重复消费、死信处理 |
| Cache | 降低重复计算或远程访问成本 | Key 设计、失效策略、一致性 |
| Scheduler | 周期任务 | 多实例重复执行、锁、可观测性 |
| CQRS | 复杂领域中的读写模型分离 | Command、Query、Event、最终一致性 |

## 学习原则

- 先明确业务问题，再选择模块或传输协议。
- 所有跨进程操作都要考虑超时、失败、重试和幂等。
- 进程内事件不能天然替代可靠消息队列。
- 引入新基础设施时，同时增加健康检查、监控和测试方案。

这些主题暂未塞入基础 Demo，避免掩盖核心请求链路。后续可以在 `examples/` 中为每种能力建立独立、最小的可运行示例。
